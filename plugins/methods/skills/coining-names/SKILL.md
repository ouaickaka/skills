# Coining Names

Generate names by procedure, not free association: map the thing's attributes
to sounds, generate candidates through distinct word-formation strategies,
then screen every finalist. A name list without sound reasoning and a
screening pass is a mood board, not naming work.

## Procedure

### 1. Extract attributes before generating

List 3-5 attributes the name must evoke (fast, safe, friendly, precise,
heavy-duty…) and the register (playful/technical/premium). Every candidate
later cites which attributes it encodes.

### 2. Map attributes to sounds

Sound symbolism is measurable and cross-linguistic — use it deliberately:

| Sound class | Evokes |
|---|---|
| Plosives k, t, p, b, d, g | speed, impact, strength, precision |
| Front vowels i, e | small, quick, light, sharp |
| Back vowels o, u | large, heavy, slow, deep |
| Liquids/nasals l, r, m, n | smooth, soft, premium, calm |
| Fricatives s, z, f, v | motion, air, friction, energy |
| Voiced vs voiceless | voiced (b, d, z) reads heavier/rounder than voiceless (p, t, s) |

Pick target sound classes from the attributes, then generate toward them.
"Kwikt" and "Lumo" are not interchangeable moods — they are different
phonetic claims.

### 3. Generate through at least three formation strategies

Label each candidate with its strategy:

* **Root mining (etymology):** Latin/Greek/Norse/other roots carrying the
  meaning — *celer* (swift), *arx* (citadel), *nid* (nest). Coin from the
  root, don't transliterate it.
* **Affixation (morphology):** root + productive prefix/suffix — -ly, -io,
  -ex, -ora, re-, un-. Keeps names parseable and spellable.
* **Blending:** splice two meaning-bearing words at a shared phoneme
  (Pinterest = pin + interest). Blend at a syllable boundary or it becomes
  unpronounceable.
* **Clipping/respelling:** truncate or respell a real word (Tumblr, Flickr) —
  flag that respellings cost spellability.
* **Coined/abstract:** pure phonosemantic invention (Kodak) — strongest
  trademark position, weakest built-in meaning; justify entirely by sound.

### 4. Screen every finalist

For each surviving candidate check and report:

* **Pronounceability:** obvious pronunciation on first read? Ambiguous
  vowels ("Xela") and consonant clusters kill word-of-mouth.
* **Cross-language hazards:** check the candidate against major languages
  for embarrassing or negative meanings; state which languages you actually
  checked. This screen catches the classic failures (Nova/no-va class).
* **Collision:** similar-sounding existing products in the same category
  you're aware of — name them.
* **Spellability:** can someone who heard it type it? One canonical
  spelling?

### 5. Label what you cannot verify

You cannot check trademark registers, domain availability, or app-store
collisions from your head. Never state a name is "available". Put these in a
**Needs manual verification** section: trademark search, domain lookup,
native-speaker review for target markets.

## Output

A shortlist (5-10) where every entry has: name, formation strategy,
attribute→sound rationale, screening notes — then the needs-verification
list. Recommend 2-3, and say why in terms of the attribute mapping, not
taste.

## Scope boundary

This skill names things for humans (brands, products, projects). Identifier
naming inside code follows the codebase's conventions, not this procedure.
